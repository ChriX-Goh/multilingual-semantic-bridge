# Multilingual Semantic Bridge — Public Copy Pack

## One-line
A skill that helps non-English or mixed-language technical requests hit the right English-heavy targets more reliably.

## Short
Multilingual Semantic Bridge improves retrieval and routing when user phrasing and technical targets do not naturally live in the same language. It preserves the original request, derives canonical intent, adds an English technical pivot when useful, and helps the assistant reach the right memory, docs, skills, files, and exact technical surfaces.

## Medium
Multilingual Semantic Bridge is a cross-lingual retrieval and routing skill for technical environments where user language and system terminology differ. Instead of acting like a generic translator, it works as a target-matching layer: preserve original input, derive canonical intent, generate an English technical pivot when needed, bridge terminology, and improve routing toward memory, official docs, skill metadata, local files, config keys, and exact-token technical surfaces.

It is especially useful when the user asks in Chinese or another non-English language, but the most useful targets are named in English-heavy operational or product language.

## Long
Multilingual Semantic Bridge helps assistants operate across a language mismatch that is common in technical environments: the user may ask in one language, while the most useful targets—official docs, config keys, provider names, CLI commands, skill metadata, and operational files—are often named in another, usually English.

The skill does not reduce that problem to a phrasebook or a translation-only workflow. Instead, it treats the problem as target resolution. It preserves the original request, derives canonical intent, generates an English technical pivot when that improves target matching, bridges terminology across user wording and official terminology, and improves retrieval/routing toward the correct technical surface.

This makes it useful for multilingual memory recall, documentation lookup, skill discovery, local operational-file targeting, and technical error/config interpretation where exact terms matter.

## Boundary copy
This skill improves retrieval and routing discipline. It does not by itself guarantee better embeddings, backend-level retrieval improvements, or universal multilingual performance across every domain.

## Best-fit bullets
- multilingual memory recall
- official-doc lookup from non-English requests
- skill discovery when skill metadata is English-heavy
- local operational-file targeting
- config/log/error interpretation where exact English tokens matter
