## 3. One-line description

## 4. Short public description
- A bridge skill for technical environments where user phrasing and the most useful target terminology do not naturally live in the same language. It preserves original wording, derives canonical intent, uses an English technical pivot when useful, and improves routing toward memory, docs, config terms, skill metadata, and exact technical surfaces.

## 5. Core value points
- not a translation-only skill
- not a per-language hard-coded rule pack
- focuses on target matching and routing precision
- useful for memory, docs, config, and operational technical surfaces

## 6. Honest boundary statement

- Does not by itself guarantee backend retrieval upgrades, perfect multilingual reasoning, or universal coverage.
